# Week 1 Build Log —— Monad & Web3 入门实践记录

## 一、Week 1 学习目标

本周主要目标是完成 Web3 基础环境搭建，并通过实际操作理解链上钱包、测试网交易、智能合约部署以及 AI 辅助开发流程。

通过这一周的实践，我从最初只了解区块链概念，逐渐开始接触真实的链上操作流程，包括钱包创建、测试网连接、交易确认、区块浏览器查询以及 Solidity 合约部署。

# 二、链上实践记录

## 1. 钱包配置与 Monad Testnet 接入

本周完成了 MetaMask 钱包配置，并成功添加 Monad Testnet 网络。

使用工具：

- 钱包：MetaMask
- 网络：Monad Testnet
- 测试资产：MON Token

完成内容：

- 创建并配置 Web3 钱包
- 添加 Monad Testnet 网络
- 获取测试网 MON 资产
- 查看钱包余额和网络状态

通过这一步，我理解了 Web3 钱包不仅是资产存储工具，也是用户进入链上生态的重要入口。

---

## 2. 完成第一笔 Monad Testnet 链上交易

完成了一次 Monad Testnet 链上交易，并通过区块浏览器查看交易状态。

交易过程中重点学习：

- from：交易发起地址
- to：交易接收地址
- value：转账金额
- gas：执行交易所消耗的手续费

交易流程：

1. 使用 MetaMask 发起交易。
2. 等待网络确认。
3. 获取 Transaction Hash。
4. 在区块浏览器查看交易详情。

通过实际交易，我理解了区块链交易与传统互联网请求的区别。

传统应用的数据通常存储在中心化服务器，而链上交易需要经过网络节点验证，并记录在区块链中。

# 三、智能合约实践记录

## 1. 使用 AI 辅助生成 Solidity 合约

本周使用 AI 辅助生成了一个简单 Solidity 合约。

选择方向：

Message Board（留言板合约）

功能：

- 用户可以写入留言。
- 用户可以读取当前留言。

AI 使用 Prompt：
请帮我生成一个最简单的 Solidity 智能合约，实现一个留言板功能。
要求：

1.可以保存用户输入的信息。
2.提供读取信息的方法。
3.使用 Solidity 0.8.x。
4.添加简单注释帮助理解代码。

---

## 2. AI 输出内容

AI 生成了包含以下结构的 Solidity 合约：

- contract MessageBoard
- string 类型变量保存留言
- writeMessage() 写入函数
- readMessage() 查询函数

代码逻辑：

用户调用 writeMessage 函数，将新的留言写入区块链。

用户调用 readMessage 函数，可以读取当前保存的信息。

---

## 3. 人工检查与修改

虽然 AI 可以快速生成代码，但仍需要人工判断。

我主要检查：

### （1）合约是否能够正常编译

确认 Solidity 版本和语法没有错误。

### （2）函数逻辑是否符合需求

确认：

- 写入函数能够修改状态。
- 查询函数能够正确返回数据。

### （3）安全性检查

检查：

- 是否存在明显权限问题。
- 是否存在不必要复杂逻辑。
- 变量命名是否容易理解。

经过检查后，对 AI 输出代码进行了理解和确认。

# 四、合约部署实践

使用 Remix 完成 Solidity 合约部署。

流程：

1. 在 Remix 创建 Solidity 文件。
2. 编译合约。
3. 连接 MetaMask。
4. 切换 Monad Testnet。
5. 部署 MessageBoard 合约。
6. 获取合约地址和部署 Transaction Hash。

部署完成后：

- 获得合约地址。
- 查看部署交易记录。
- 调用 writeMessage 完成一次合约交互。
- 获取交互 Transaction Hash。

---

# 五、遇到的问题与解决过程

## 问题1：Remix 找不到 MetaMask

开始使用 Remix 时，没有看到 Injected Provider。

解决：

检查 Environment 设置，将环境切换为：

Browser Extension → MetaMask

之后 Remix 成功连接钱包。

---

## 问题2：不知道 Transaction Hash 如何获取

第一次不知道在哪里查看交易记录。

解决：

在 Remix 部署或调用合约后：

- 查看交易记录区域。
- 点击交易详情。
- 获取 Transaction Hash。

同时可以复制 Hash 到 Monad 区块浏览器查询。

---

## 问题3：不理解区块浏览器作用

通过实践了解到：

区块浏览器类似区块链的数据查询工具，可以查看：

- 钱包地址
- 交易记录
- Gas 消耗
- 合约地址
- 区块信息

它帮助用户验证链上行为是否真实发生。

# 七、我对 Monad 和 Web3 的理解变化

通过 Week 1 实践，我对 Web3 的理解从“区块链是一种技术概念”变成了更加具体的认识。

以前认为 Web3 主要是数字资产和交易，而实际体验后发现：

用户通过钱包作为身份入口，通过智能合约完成可信交互。

Monad Testnet 的体验让我感受到，高性能公链对于游戏、社交、实时交互等场景非常重要，因为这些应用需要：

- 更快交易确认。
- 更低交互成本。
- 更好的用户体验。
