> 来听，来问，来带走一份能立刻动手的入门路线图。

### 详细流程

#### 0:00–0:03 | 暖场（主持人）

**目标：** 让听众知道「这场 Space 适合我」，建立规则预期。

**主持人串场要点：**

- 欢迎语 + 自我介绍（一句话）。
- 明确本场 Space 的定位：**零基础友好，不讲学术，只讲怎么动手**。
- 说明互动规则：前 35 分钟嘉宾对谈，后面有互动环节和自由问答，想提问的听众可以在评论区提前留言。
- 一句话预告：活动结束后会发放「AI Agent 零基础入门资源包」。

> **串场示范：**
> 「欢迎来到今天的 Space。如果你总觉得 AI Agent 听起来很厉害但不知道从哪开始，如果你一直以为这东西需要写代码才能碰——那今天这场就是专门为你准备的。我们会用 70 分钟，让你从『AI Agent 是什么』，走到『我知道怎么开始』。最后还会发一份资源包，里面有工具推荐、学习路径和社区入口。好，先请三位嘉宾跟大家打个招呼。」

---

#### 0:03–0:08 | 破冰：嘉宾自我介绍（主持人引导）

**目标：** 快速建立嘉宾信任感，让听众产生「他们也是从零开始的」的共鸣。

**形式：** 每位嘉宾 1 分钟自我介绍 + 用一句话定义「AI Agent 是什么」。

| 嘉宾 | 角色 | 自我介绍重点 |
|------|------|-------------|
| 嘉宾 A | AI Agent 开发者 | 什么时候开始学、现在在做什么、一句话证明「零基础也能学会」 |
| 嘉宾 B | 零基础转型新人 | 之前的职业、转型契机、第一个 Agent 做了什么 |
| 嘉宾 C | Web3 Builder | 为什么 Web3 开发者需要关心 AI Agent、正在做的项目 |

> **设计意图：** 三位嘉宾分别代表「技术深度」「转型故事」「交叉领域」三条叙事线，让不同类型的听众都能找到认同对象。同时「一句话定义 AI Agent」制造认知碰撞——三个人给出三个视角，比主持人直接讲定义更有说服力。

---

#### 0:08–0:35 | 主题对话：三幕式对谈（主持人引导）

**目标：** 用 27 分钟完成「认知建立 → 路径交付 → 价值说服」的完整叙事弧线。

---

##### 第一幕：是什么（0:08–0:16，8 分钟）

**核心问题 1：AI Agent 到底是什么？和 ChatGPT 有什么区别？**

**主持引导方向：**

- 嘉宾 A 用生活化场景解释：ChatGPT 是「你问它答」，AI Agent 是「你给它一个目标，它自己规划步骤、调用工具、完成交付」。
- 嘉宾 B 补充：自己当初最困惑的就是这个区别，直到用了一个类比才理解——「ChatGPT 是图书馆管理员，AI Agent 是你的私人助理」。
- 嘉宾 C 从 Web3 视角补充：链上 Agent 可以自动监控链上数据、执行交易策略，这是普通聊天机器人做不到的。

> **此环节目标：** 8 分钟内让零基础听众建立起「AI Agent ≠ 聊天机器人」的核心认知。

---

##### 第二幕：怎么做（0:16–0:27，11 分钟）

**核心问题 2：我需要学多少编程才能开始？**

**核心问题 3：我第一次应该做什么？有没有一个最小可完成项目？**

**主持引导方向：**

- 嘉宾 A 展示三条路径的对比：
  - **无代码路径**（Fleece AI、Zapier AI、Botpress）：适合零编程背景，10 分钟出结果。
  - **低代码路径**（Coze、Dify、Relevance AI）：有一点逻辑思维即可，功能更灵活。
  - **代码框架路径**（LangChain、CrewAI、Mastra）：给有编程基础的人，灵活性最高。
- 嘉宾 B 分享自己「第一个 Agent」的具体故事：做了什么、用了什么工具、花了多长时间、踩了什么坑。
- 三位嘉宾各推荐一个「最小可完成项目」作为听众的起点，例如：
  - 嘉宾 A：一个自动整理 Web3 日报并推送到 Telegram 的 Agent
  - 嘉宾 B：一个帮你总结 X 平台热门推文的情感分析 Agent
  - 嘉宾 C：一个监控指定钱包地址异动并发送提醒的链上 Agent

> **此环节目标：** 给听众三条清晰路径 + 三个可立刻动手的项目选项，消除「不知道做什么」的迷茫。

---

##### 第三幕：为什么值得（0:27–0:35，8 分钟）

**核心问题 4：学了能做什么？有什么实际价值？**

**核心问题 5：Web3 和 AI Agent 结合的未来是什么？**

**主持引导方向：**

- 嘉宾 C 分享 AI Agent 在 Web3 中的真实应用场景：链上数据分析、自动化 DeFi 策略、AI 驱动的 DAO 治理、去中心化 Agent 网络。
- 嘉宾 A 谈市场趋势：2026 年 AI Agent 开发者需求正在爆发，企业端 Agent 实战培训已经覆盖 600+ 企业（参考网易智企案例）。
- 嘉宾 B 谈个人收益：学会 AI Agent 后对自己的职业或副业带来的实际变化。
- 三人简短展望：2026–2027 年，AI Agent + Web3 的结合会出现什么新机会？

> **此环节目标：** 让听众从「听听看」转向「我要学」——用真实需求和市场前景完成价值说服。

---

#### 0:35–0:50 | 互动环节：「30 秒电梯路演」（主持人引导）

**目标：** 把听众从「被动接收」变成「主动参与」，制造活动的高潮记忆点。

**形式：**

1. 主持人公布互动规则：听众举手（或评论区自荐），获得 30 秒时间，用一句话描述「如果我能做一个 AI Agent，我希望它帮我解决什么问题」。
2. 挑选 3–5 位听众发言。
3. 每位听众发言后，嘉宾中选一位给 1 分钟即时反馈——这个想法能不能用现有工具实现、大概需要什么路径。
4. 全部发言结束后，三位嘉宾各投一票选出「最有创意 idea」，获奖者将获得一份额外的 1v1 入门指导（30 分钟线上）。

**主持人串场要点：**

- 鼓励零基础听众大胆发言：「不需要你的 idea 很技术，只要是你真正想解决的问题。」
- 控制节奏：每人 30 秒发言 + 1 分钟嘉宾反馈，5 位听众约 8 分钟，加上规则说明和投票约 15 分钟。
- 如果举手人数不够，主持人可以抛出几个备选 prompt 引导：「比如，有没有人想做一个帮你自动回复 Discord 消息的 Agent？或者帮你分析某个 NFT 项目风险的 Agent？」

> **设计意图：**
> - **降低参与门槛：** 不需要技术背景，只需要说出「想解决什么问题」。
> - **即时正反馈：** 嘉宾即时的可行性反馈让听众感受到「原来我的想法真的能实现」。
> - **社交货币：** 被选中的听众在其他参与者面前曝光，产生归属感。
> - **制造 FOMO：** 没发言的人看到别人获得即时反馈，会更积极参与后续活动。

---

#### 0:50–1:05 | 开放问答（主持人引导）

**目标：** 解决个性化问题，弥补主题对话覆盖不到的盲区。

**形式：**

- 主持人从评论区筛选问题，按优先级排列：
  1. 被多个听众点赞的问题（代表群体需求）
  2. 与主题直接相关但未被覆盖的问题
  3. 有趣但偏门的问题（作为收尾的轻松时刻）
- 每位嘉宾回答 2–3 个问题，主持人控制每个问题不超过 2 分钟。

**常见问题预案（供主持人参考）：**

| 预设问题 | 推荐回答嘉宾 |
|----------|-------------|
| 无代码工具做出的 Agent 和代码写的有什么区别？ | 嘉宾 A |
| 学了 AI Agent 能找到工作吗？薪资大概多少？ | 嘉宾 B + 嘉宾 A |
| 我想学但怕学不会，有没有什么心理建议？ | 嘉宾 B |
| Web3 项目中真的有人在用 Agent 吗？能举几个例子？ | 嘉宾 C |
| 有没有免费的入门资源推荐？ | 嘉宾 A |

---

#### 1:05–1:10 | 收尾 CTA（主持人）

**目标：** 让听众带着明确的下一步行动离开，而不是「听完就忘了」。

**主持人串场要点：**

1. **活动回顾（1 分钟）：** 一句话总结今天的三幕内容——「我们知道了 AI Agent 是什么，知道了三条入门路径，也看到了它在 Web3 中的真实价值。」
2. **资源包发放（1 分钟）：** 告知「AI Agent 零基础入门资源包」的获取方式——
   - 关注主持人/嘉宾的 X 账号，资源包链接将在 Space 结束后 24 小时内通过推文发布。
   - 资源包内容：工具推荐清单（含无代码/低代码/代码框架）、三条路径的学习路线图、推荐社区与社群入口、今日嘉宾推荐的「第一个 Agent 项目」操作指南。
3. **核心 CTA（2 分钟）：** 三个明确的下一步行动——

   > **CTA 1：立刻动手。** 今天选一个工具（推荐 Fleece AI 或 Coze），花 30 分钟试着搭建一个最简单的 Agent——哪怕只是「自动回复一句早安」。做了，你就已经超过了 90% 的观望者。
   >
   > **CTA 2：加入社群。** 加入我们的 Discord/Telegram 群组（链接见资源包），在群里分享你的第一个 Agent 截图。我们会在群里做答疑和反馈。
   >
   > **CTA 3：参加下一场。** 两周后我们将举办第二场 Space——「AI Agent 实战：从你的第一个项目到上线」。我们会邀请今天互动环节的获奖者来分享他的 Agent 进展。
   >
   > **CTA 4：分享收获。** 如果你觉得今天的内容对你有帮助，发一条推文标记我们，分享你最大的一个收获。我们会从前 20 条推文中抽 3 位送出一对一入门指导。

4. **结束语（1 分钟）：** 感谢嘉宾、感谢听众，预告下一场活动时间，道别。

> **设计意图：**
> - CTA 1 强调「立刻行动」——降低心理门槛，30 分钟是任何人都能承诺的时间。
> - CTA 2 提供社群归属——独自学习的放弃率很高，社群是留存的关键。
> - CTA 3 制造期待——让听众知道「这不是一次性活动」，形成系列化品牌认知。
> - CTA 4 激励传播——用社交分享扩大活动影响力，为下一场引流。

## 五、互动环节设计

### 「30 秒电梯路演」

**环节名称：** 如果你的 AI Agent 能帮你做一件事

**时间：** 15 分钟

**规则：**

1. 听众举手或被主持人邀请，获得 30 秒发言时间。
2. 发言内容：用一句话描述「如果我能做一个 AI Agent，我希望它帮我解决什么问题」。
3. 嘉宾选一位给 1 分钟即时反馈——这个想法能不能用现有工具实现、大概需要什么路径。
4. 选择 3–5 位听众发言。
5. 全部发言结束后，三位嘉宾各投一票选出「最有创意 idea」。
6. 获奖者获得 30 分钟 1v1 入门指导。

**备选方案（如果举手人数不足）：**

主持人抛出 3 个场景 prompt，请听众在评论区投票选择最感兴趣的方向，然后邀请几位投票者语音分享：

- 「帮你自动回复 Discord/Telegram 社区消息的 Agent」
- 「帮你分析某个 NFT 项目风险的 Agent」
- 「帮你每天整理 Web3 行业动态并推送的 Agent」
