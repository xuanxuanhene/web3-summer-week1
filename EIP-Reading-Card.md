## 背景问题

在 ERC-20 出现之前，以太坊已经支持开发者发行 Token，但没有统一的接口标准。不同项目会使用不同的函数名称和实现方式，例如转账、查询余额、授权等功能都可能各不相同。

这导致钱包、交易所和 DApp 在支持新的 Token 时，需要针对每个项目单独开发兼容逻辑，增加了开发成本，也影响了整个生态的互操作性。因此，以太坊社区提出了 ERC-20，希望建立一个统一的同质化 Token 标准。

---

## 核心方案（自己的理解）

我认为 ERC-20 的核心思想不是发明一种新的 Token，而是规定了一套所有 Token 都应该遵循的统一接口。

标准要求 Token 至少实现以下几个核心函数：

- `transfer()`：实现 Token 转账。
- `balanceOf()`：查询指定地址的 Token 余额。
- `approve()`：授权其他地址使用自己的 Token。
- `transferFrom()`：授权后的代理转账。
- `allowance()`：查询剩余授权额度。

此外，还规定了 `Transfer` 和 `Approval` 两个事件，方便钱包、浏览器和其他应用监听 Token 状态变化。

这样一来，只要一个 Token 遵循 ERC-20 标准，整个生态中的钱包、交易所和 DeFi 协议就可以直接兼容它，而无需重复开发。

---

## 影响对象

### 普通用户

可以在不同钱包和 DeFi 应用中，以一致的方式使用各种 ERC-20 Token，体验更加统一。

### 钱包开发者

只需要支持 ERC-20 标准，即可兼容大量 Token，大幅降低开发和维护成本。

### 交易所

中心化交易所和去中心化交易所都能更快接入新的 ERC-20 Token，提高上线效率。

### DeFi 协议

借贷、DEX、稳定币等协议都建立在 ERC-20 标准之上，实现了资产之间的自由组合。

### Token 开发者

只需遵循 ERC-20 标准，就能够让自己的 Token 更容易进入整个 Ethereum 生态。

## 关键术语

- **ERC（Ethereum Request for Comments）**：以太坊标准提案。
- **Fungible Token（同质化 Token）**：每个 Token 都具有相同价值，例如 1 USDT = 1 USDT。
- **Smart Contract（智能合约）**：ERC-20 Token 的实现方式。
- **approve（授权）**：允许其他地址使用自己的 Token。
- **allowance（授权额度）**：剩余可使用的授权数量。
- **transferFrom（代理转账）**：授权后代表用户完成 Token 转移。
- **Transfer Event**：记录 Token 转账事件。
- **Approval Event**：记录授权事件。

## 争议与风险

虽然 ERC-20 极大推动了 Ethereum 生态的发展，但也存在一些问题。

1. **无限授权风险（Unlimited Approval）**

很多用户为了方便，会一次性授权无限额度。如果协议被攻击或存在漏洞，攻击者可能直接转走用户全部 Token。

2. **Approve Race Condition**

修改授权额度时可能存在竞争条件，因此后来很多钱包建议先将额度设为 0，再重新授权。

3. **误转 Token**

如果 ERC-20 Token 被发送到不支持 ERC-20 的合约地址，通常无法找回。

4. **标准较简单**

ERC-20 只规定接口，没有规定冻结、黑名单、手续费等功能，因此不同项目会自行扩展，造成一定差异。

---

## 产品启发

阅读 ERC-20 后，我认为最大的启发有以下几点：

- **标准化比增加功能更重要。** 一个统一的接口能够降低整个生态的开发成本。
- **开放标准能够形成网络效应。** 越多项目采用同一标准，生态价值越高。
- **产品设计应注重可组合性。** ERC-20 让钱包、DEX、借贷协议等能够相互协作，推动了 DeFi 的快速发展。
- **基础设施的成功往往来自兼容性，而不是复杂性。** ERC-20 本身并不复杂，但成为了整个以太坊生态的重要基础。

## 我的疑问

阅读提案后，我还有一些问题没有完全理解：

1. 为什么后来提出的 ERC-777、ERC-223 等标准没有取代 ERC-20？
2. 是否能够设计一种默认更加安全、减少无限授权风险的新 Token 标准？
3. 在多链生态下，ERC-20 如何与其他公链的 Token 标准实现更好的互操作？

---

## 阅读总结

通过阅读 EIP-20 原文，我了解到 ERC-20 最重要的贡献并不是发明了 Token，而是建立了一套统一的 Token 标准。这套标准降低了开发者的接入成本，提高了钱包、交易所和 DeFi 协议之间的兼容性，也促进了整个 Ethereum 生态的发展。我认为，ERC-20 是 Web3 世界中标准化和可组合性设计思想的典型代表。
