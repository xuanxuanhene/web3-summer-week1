# Moss：连接 AI Agent 与 Web3 世界的开源探索

## 项目背景

随着大语言模型（LLM）和智能体（AI Agent）技术的发展，AI 应用正在从传统的信息交互模式，逐渐走向具备自主决策、任务执行和工具调用能力的新阶段。

与此同时，Web3 生态正在推动去中心化应用、数字资产和开放协议的发展。

然而，AI Agent 与 Web3 之间仍存在一定技术鸿沟：

* AI Agent 缺少访问区块链生态的标准化方式；
* Web3 应用缺少智能化交互能力；
* 开发者需要重复实现钱包管理、链上调用、协议连接等基础功能。

Moss 的目标正是在 AI Agent 与 Web3 之间建立连接，让开发者能够更加方便地构建具备 Web3 能力的智能 Agent。

项目地址：

https://github.com/nishuzumi/moss

---

# Moss 是什么？

Moss 是一个面向 **AI Agent × Web3** 场景的开源项目。

它希望帮助开发者构建能够理解用户需求，并进一步与区块链生态交互的智能 Agent。

相比传统 AI 助手只能完成文本生成和信息分析，结合 Moss 的 Agent 可以探索更多能力：

* 调用 Web3 服务；
* 获取链上信息；
* 与去中心化应用交互；
* 执行自动化任务；
* 连接开放式数字生态。

简单来说：

> Moss 希望成为 AI Agent 进入 Web3 世界的连接层。

---

# 为什么 AI Agent 需要 Moss？

## 1. 从“回答问题”到“执行任务”

传统 AI 应用主要完成：

```
用户输入 → AI 理解 → 输出回答
```

而未来的 AI Agent 需要：

```
用户需求
    ↓
AI 分析任务
    ↓
调用工具
    ↓
访问 Web3 服务
    ↓
完成实际操作
```

例如：

用户希望寻找最佳 DeFi 策略，AI Agent 不仅可以提供分析，还可以进一步获取链上数据、比较协议状态，并在用户授权后执行相关操作。

这要求 AI Agent 具备连接 Web3 世界的能力。

开发者可以构建：

* 项目分析 Agent；

* 智能合约研究助手。

帮助用户降低理解 Web3 信息的成本。

## DAO 与社区 Agent

未来的去中心化组织可能需要大量智能 Agent：

* 自动整理治理信息；
* 分析社区提案；
* 辅助成员决策；
* 执行公开任务。

Moss 为这些开放式应用提供了一种探索方向。
